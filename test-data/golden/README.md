# LawDesk Golden Dataset V1.0

This directory contains manually curated golden cases for repeatable LawDesk workflow validation.

The golden dataset is not generated by an AI provider. Model output belongs in `runs/<timestamp>/actual/`; expected answers remain human-reviewed standards.

## Scope

- V1 includes `case01-private-lending` only.
- V1 validates the frozen workflow from Matter Draft through Document Draft.
- Legal rule verification is tracked separately in each case's `review/` directory.

## Directory Convention

```text
test-data/golden/
  dataset.json
  scoring.json
  case01-private-lending/
    case.json
    materials/
    expected/
    review/
    runs/
```

## Commands

LawDesk V1 uses only `scripts/golden/` with this `test-data/golden/` dataset contract:

```bash
pnpm golden:run case01-private-lending --provider=mock
pnpm golden:run case01-private-lending --provider=minimax
pnpm golden:validate case01-private-lending
pnpm golden:report case01-private-lending
```

`golden:run` requires the caller safety declaration `GOLDEN_DATABASE_NAME=lawdesk_rc_test`. This records the caller-declared target database; it does not independently verify the Backend database connection. The runner creates a unique Matter and never invokes a global reset or purge endpoint.

## Run Output

Each run should write:

```text
runs/<timestamp>/
  run.json
  actual/
    evidence.json
    facts.json
    issues.json
    laws.json
    arguments.json
    document.json
    relations.json
  report.json
  report.md
```

Run output must not include API keys, full prompt text, or sensitive environment variables.
