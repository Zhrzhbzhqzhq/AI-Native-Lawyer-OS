# Case03 AI Quality Review

## 案件类型

商品房买卖合同纠纷，核心事项为房屋延期交付、继续履行、合同解除及违约责任。

本次真实运行对应 Matter：`m-mryqi0y7-t1e3sq`。案件处于诉前咨询、催告阶段，尚未形成可直接提交法院的最终诉讼方案。

## 测试目标

本 Golden Dataset 用于验证 LawDesk V1 能否基于当前 Matter 的真实材料，连续完成以下案件认知链：

1. 识别案件类型、主体、案件阶段和核心争议。
2. 将 Material 整理为可供律师审核的 Evidence。
3. 建立 Evidence → Fact → Issue → Law → Argument 的可追溯关系。
4. 基于正式来源生成民事起诉状 Document Draft。
5. 避免跨 Matter 污染、错误案件类型和材料外事实。

本测试不用于证明 AI 已经能够替代律师确定诉讼策略、确认法律结论或直接提交文书。

## 输入材料说明

Golden Dataset 中包含以下七组真实测试材料，并保留 DOCX、PDF 版本用于文件读取验证：

- 客户咨询记录
- 商品房买卖合同
- 付款记录
- 微信沟通整理
- 房屋交付催告函
- 被告答复
- 客户诉求

现有材料能够确认合同主体、房屋总价、约定交付期限、两笔付款、催告内容及双方当前立场。材料没有完整呈现合同全文、房屋现状、催告送达情况、微信原始载体、付款原始流水、管辖依据及对方所称“客观因素”的具体证据。

### 已完成能力

- 正确识别为商品房买卖合同纠纷，没有出现借贷等其他案件语义污染。
- 正确识别王某与某房地产开发有限公司及双方基本立场。
- 能从七组材料形成 Case Understanding，并继续生成 Evidence、Fact、Issue、Law、Argument 和 Document Draft。
- 数据库中的正式层级已形成可追溯关系。Golden expected 校验中未发现悬空的 Evidence、Fact、Issue、Law 或 Argument 来源 ID。

- 当前输出是律师审核前的 AI 工作结果，不是已确认法律意见。
- Evidence 中包含客户陈述、沟通整理和诉求说明，这些材料的证明力与客观书证不同，仍需律师分级审核。

## Evidence 评价

### 已完成能力

- 七条 active Evidence 均关联当前 Matter 的具体 `material_id`。
- 合同、付款记录、催告函、微信沟通整理和被告答复的证明目的基本围绕商品房交付纠纷展开。
- 对客户咨询记录和客户诉求使用较为保守的证明目的，指出其属于当事人陈述或需要律师确认的内容。
- 对微信整理、催告函、被告答复等材料指出了日期、送达、签章、原始载体等形式性缺口。

### 当前限制

- 付款记录不是银行流水本身，不能仅凭整理记录最终确认付款真实性及收款主体。

- 催告函缺少发送时间、送达方式和签收证明。
- 被告答复的签章、形成时间和出具主体需要核验。
- 客户咨询记录、客户诉求主要用于案件理解和策略确认，其作为诉讼证据的资格、证明力及提交必要性应由律师决定。

限制在于：Issue 对违约金、损害赔偿、抗辩关系的讨论受到合同全文缺失影响；现阶段不能据此确定最终诉请组合。

限制在于：法条编号、条文原文、司法解释名称与现行版本、引用范围及具体适用条件必须由律师通过权威法源逐条复核。Golden 文件记录的是当前 V1 真实输出，不代表这些引用已经完成专业校验。

限制在于：部分论证篇幅较长，并混合了事实复述、法律分析和案件风险；其结论依赖尚未核验的送达、合同条款、房屋状态及法源，不能直接作为最终律师意见。

## Document Draft 评价

### 已完成能力

- 正确生成“民事起诉状”类型草稿。
- 原告、被告投影正确，没有发生跨 Matter 主体污染。
- 事实与理由使用了当前 Formal Fact、Law 和 Argument，并保存完整来源 ID。
- 没有自动生成违约金、利息、律师费或未经确认的具体金额请求。
- 法院、具状人和日期等缺失字段保留了律师补充占位符。

### 当前限制

- 当前诉讼请求只有“诉讼费用由被告承担”，缺少继续履行交付房屋或解除合同等实体请求，尚不构成完整民事起诉状。
- 草稿同时讨论继续履行和解除合同，但没有形成明确的主位、备位诉讼请求及两者之间的法律关系。
- 文书正文保留了较多内部分析、证据风险和条件判断，整体更接近案件论证工作稿，而非可直接提交的法院文书。
- 合同解除条件、催告送达、房屋现状、剩余房款安排、违约责任条款均未补全。
- 当前状态为 `generated`，`published_document_id` 为空，必须经过律师审核、修改和确认后才能进入正式文书流程。

## 当前版本能力结论

LawDesk V1 已能够在 Case03 中恢复完整案件语义，并形成 Material → Evidence → Fact → Issue → Law → Argument → Document Draft 的连续、可追溯链路。案件类型、主体、主要事实和核心争议总体正确，未观察到跨案件或借贷语义污染。

当前能力适合用于律师案件梳理、证据初筛、争议识别、法律方向提示和文书初稿组织。当前版本尚不能独立完成诉讼目标确定、实体诉讼请求设计、法源最终核验和可提交文书定稿。AI 输出仍然是工作建议，律师必须审核并作出最终决定。
