LawDesk Legal Reasoning Model V1.0
=================================
法律推理模型

1. LawDesk 定位

LawDesk 是面向独立律师的 AI-Native 律师操作系统（Operating System）。其目标是把案件处理的各类信息、证据、事实、法律推理与文书产出纳入可管理、可追踪、可复用的结构化流程中，减轻律师认知负担并在“人-机协作”框架下提升办案效率与质量。

2. 八层推理模型（自下而上）

- Matter（案件）：业务聚合根，承载本案所有对象与工作空间。
- Material（资料）：原始文件、上传的证据材料、合同、图片、音视频等原始资产。
- Evidence（证据）：从 Material 或其他来源经确认、可用于证明事实的条目（例：转账记录、聊天记录、证人陈述）。
- Fact（事实）：基于 Evidence 提炼的客观事实陈述（如“2023-01-10 付款 100000 元”）。
- Issue（法律问题）：律师需要回答或解决的法律/程序/策略问题，建立在 Fact 之上并指向法律适用与论证方向。
- Law（法律）：适用的法律条文、司法解释与判例等法律资源，用于支撑对 Issue 的法理推导。
- Argument（论证）：根据 Facts 与 Law 对 Issue 的推理链、证据评价与法律适用结论（包括支持与反对意见、风险点与证据缺口）。
- Document（文书）：基于 Argument 输出的可交付物（起诉状、代理词、意见书、保全申请等）。

3. 每一层职责（简要）

- Matter：案件级别的权限、元数据、聚合接口与工作空间划分。
- Material：保存原始证据资产与元信息，负责文件来源追溯与版本管理。
- Evidence：证据的确认、类型、相关性与初步评价（是否可作为举证材料）。
- Fact：归纳可证明的事实，作为法律推理的原材料（不承担法律判断）。
- Issue：定义需要解决的问题，列出问题的范围、优先级与分析目标。
- Law：存储相关条文、司法解释与要点摘录，用于规则匹配与引用。
- Argument：组织事实与法律的结合，形成可审查的推理链与结论备选项。
- Document：将 Argument 模板化并生成可提交/传阅的文书文本。

4. 各层之间的关系（关键约束）

- Data flow（自下而上）：Material → Evidence → Fact → Issue → Argument → Document。Law 在 Issue/Argument 阶段被引用并参与推理。
- 关联关系：
  - Evidence 与 Fact：多对多（同一证据可证明多个事实；一个事实可由多证据支持），通过 FactEvidence 关联。
  - Fact 与 Issue：多对多（同一事实可支撑多个问题；一个问题可引用多个事实），建议通过 IssueFact 关联表并保留注释字段说明支持方式。
  - Issue 与 Law：多对多（同一法律条文可支撑多个 Issue；一个 Issue 可引用多条法律依据），建议建立 IssueLaw 引用表以存储引用片段/理由。
  - Argument 关联 Fact/Issue/Law：Argument 为汇聚体，需可引用 Fact、Issue、Law 并包含作者、版本与论证文本。

5. AI 与律师的分工（原则性说明）

- AI 的角色（辅助、不可替代决策者）：
  - 信息抽取：从 Material 中提取证据候选、OCR、结构化转账/时间/参与人信息（提高数据录入效率）。
  - 证据聚合与初步评价：根据规则或模型提示证据关联的 Fact 候选与相关性得分（供律师参考）。
  - 法律检索与参考：快速检索可能适用的法律条文与判例，生成候选引用列表。
  - 草拟 Argument/Document 草稿：在律师监督下生成初稿供编辑，不做最终法律结论。
- 律师的角色（人类核心决策者）：
  - 事实确认：审核 AI 提取的 Fact 与 Evidence 的有效性与法律意义。
  - 法律判断：决定某一 Issue 的解决方案、适用法律与策略选择（AI 仅提供建议）。
  - 审核与签署：对 Argument 与 Document 的法律结论与文本负责并签署。

6. V2 后续开发路线（高层建议）

- M60（Issue）优先：实现 Issue 模型、IssueFact 关联与基础 UI（捕捉问题、分配、状态/优先级、与 Fact 的绑定）。
- Law 模块：建立 Law/Statute/Case 模型与检索接口，支持 Issue ↔ Law 的引用管理。
- Argument 层：设计 Argument 模型（引用 Facts、Law，存作者、版本、结论），并在 UI 提供论证编辑与审阅流程。
- Document 生成：基于 Argument 模板化输出文书草稿，支持律师编辑与版本管理。
- 增值功能（后续）：自动化建议（非替代性 AI 建议）、证据缺口提示、Issue 工作流与任务集成、审阅/签署日志。

7. 明确说明（关于 Issue 的定义）

- Issue = 法律问题（Legal Issue），但不限于狭义的法律条文适用问题。Issue 的范围包括：程序问题（如证据保全是否必要）、策略问题（起诉 vs 和解）、执行问题（如何保全财产）等。Issue 是律师要解决的事项集合，是业务与法律判断的结合体。
