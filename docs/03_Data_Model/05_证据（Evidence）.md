# Evidence（证据）数据模型

## 一、定义

Evidence 是案件中的证据对象。

证据是律师办案的核心材料之一，AI Native Lawyer OS 需要帮助律师完成证据上传、分类、识别、整理、分析和缺失提醒。

## 二、基本字段

| 字段 | 含义 | 示例 |
|---|---|---|
| evidence_id | 证据编号 | E-2026-0001 |
| matter_id | 关联案件编号 | 2026-0001 |
| evidence_name | 证据名称 | 借条 |
| evidence_type | 证据类型 | 合同 / 转账记录 / 聊天记录 / 录音 / 视频 / 图片 / 判决书 / 其他 |
| source | 证据来源 | 客户提供 / 律师收集 / 法院材料 / 对方提交 |
| file_path | 文件路径 | /案件资料/证据/借条.pdf |
| file_format | 文件格式 | pdf / docx / jpg / mp3 |
| submitted_by | 提交人 | 张三 |
| submit_date | 提交日期 | 2026-06-30 |
| evidence_status | 证据状态 | 已上传 / 已识别 / 待确认 / 已采用 / 不采用 |
| proof_purpose | 证明目的 | 证明双方存在借贷关系 |
| related_fact | 关联事实 | 2025年5月1日借款10万元 |
| ai_summary | AI证据摘要 | 借条载明借款金额10万元，约定还款日期为2025年12月31日 |
| ai_risk | AI风险提示 | 未见李四身份证号，签名真实性需核实 |
| lawyer_note | 律师备注 | 需客户提供原件 |

## 三、证据类型

V1 支持以下类型：

1. 合同/协议
2. 借条/欠条
3. 转账记录
4. 微信聊天记录
5. 短信记录
6. 电话录音
7. 视频
8. 图片
9. 身份材料
10. 法院材料
11. 对方提交材料
12. 判决/裁定/调解书
13. 其他

## 四、证据状态

| 状态 | 说明 |
|---|---|
| 已上传 | 文件已进入案件 |
| 已识别 | AI 已完成基础识别 |
| 待确认 | 等待律师确认是否采用 |
| 已采用 | 已进入正式证据清单 |
| 不采用 | 当前不作为证据使用 |
| 需补充 | 证据不完整，需要补充材料 |
| 有风险 | 证据存在真实性、关联性、合法性或证明力风险 |

## 五、AI 可读取内容

AI 可以读取：

- 证据文件内容
- OCR 识别结果
- 录音转文字
- 图片文字识别
- 律师备注
- 关联事实
- 已有证据目录

## 六、AI 可生成内容

AI 可以生成：

- 证据摘要
- 证明目的
- 证据分类
- 证据目录草稿
- 证据链分析
- 缺失证据提醒
- 证据风险提示
- 举证意见草稿

## 七、律师确认规则

以下内容必须由律师确认后正式生效：

- 是否采用该证据
- 证明目的
- 证据目录
- 举证意见
- 证据风险判断
- 是否提交法院

## 八、Evidence 与 Matter 的关系

一个 Matter 可以包含多个 Evidence。

每个 Evidence 必须关联一个 Matter。

Evidence 可以关联：

- Timeline：案件时间轴
- Document：文书
- Task：案件任务
- AI_Work_Record：AI 工作记录

## 九、V1 最小可用字段

第一版开发至少需要：

- evidence_id
- matter_id
- evidence_name
- evidence_type
- file_path
- evidence_status
- proof_purpose
- related_fact
- ai_summary
- ai_risk
- created_at
- updated_at

## 十、设计原则

Evidence 不是简单文件，而是案件事实和法律主张的支撑材料。

AI 的核心任务不是“存证据”，而是帮助律师理解证据、组织证据、发现缺失证据，并形成可用于办案的证据体系。
